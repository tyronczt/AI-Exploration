# 工作日报/周报/月报生成 Skill

> 从 git 提交记录自动生成领导可读的工作日报，支持周报、月报聚合，告别流水账。

[![skills.sh](https://skills.sh/b/tyronczt/AI-Exploration)](https://skills.sh/tyronczt/AI-Exploration)

## 一键安装

```bash
npx skills add tyronczt/AI-Exploration/daily_skill
```

## 功能特性

- **日报生成**：从 git 仓库提取当日 commit，按业务故事线聚合，输出 Markdown 日报
- **周报聚合**：从已生成的日报文件中跨日合并故事线，生成周报
- **月报聚合**：从已生成的周报文件中跨周合并故事线，生成月报
- **自适应语义分析**：先读 commit message + stat，对模糊 commit 深入读 diff
- **噪音过滤**：自动跳过编译报错、格式调整等无业务价值的提交
- **交互式补全**：询问完成状态、协作上下文、待开发计划
- **多仓库支持**：支持同时从多个 git 仓库提取 commit
- **远程归档**：可选上传至私有 Git 仓库进行归档

## 使用方式

安装后在 Claude Code 中通过以下方式触发：

| 模式 | 触发方式 | 数据源 | 输出 |
|------|----------|--------|------|
| 日报 | `/daily-report` 或 "写日报" | git commit | `YYYY-MM-DD-工作日报.md` |
| 周报 | "写周报" | 日报文件 | `YYYY-WXX-工作周报.md` |
| 月报 | "写月报" | 周报文件 | `YYYY-MM-工作月报.md` |

## 日报示例

```markdown
# 2026-05-15 工作日报

## 用户中心模块优化与重构
1. 用户注册流程：重构注册校验逻辑，新增手机号重复注册精确提示，避免用户困惑
2. 登录超时处理：补接会话超时自动清理定时任务，修复多设备同时登录时 Token 失效不及时的问题
3. 权限校验：引入统一鉴权中间件保障接口访问安全，非授权请求不再误返回 500 错误 *(已完成)*
```

## 工作流程

```
输入变量 → Git 提取 → 噪音过滤 → 语义分析 → 故事线聚合 → 交互补全 → 输出
   │                                                              │
   │  仓库路径                                                      │  完成状态
   │  提交者邮箱                                                    │  协作上下文
   │  日期                                                         │  待开发计划
```

## 变量配置

| 变量 | 说明 | 默认值 |
|------|------|--------|
| 仓库路径 | 支持逗号分隔多个路径 | 交互式询问 |
| 提交者邮箱 | git commit 的 author | 交互式询问 |
| 日期 | 目标日期 | 今天 |
| 日报存放目录 | 日报文件存储位置 | 交互式询问 |
| 归档仓库地址 | Git 远程仓库 URL | 交互式询问 |
