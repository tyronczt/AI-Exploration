# 地铁打卡记录小程序启动说明

## 项目结构

```
subway_records/
├── front/              # 微信小程序前端代码
├── back/src/           # Node.js后端代码
├── start-backend.bat   # 后端启动脚本
├── start-frontend.bat  # 前端启动脚本
├── 技术架构文档.md     # 技术架构文档
└── 启动说明.md         # 本文件
```

## 环境要求

### 必需软件
1. **Node.js** (v14或更高版本)
   - 下载地址: https://nodejs.org/
   - 验证安装: `node --version`

2. **MySQL** (v5.7或更高版本)
   - 下载地址: https://dev.mysql.com/downloads/
   - 或使用云数据库服务

3. **微信开发者工具**
   - 下载地址: https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

### 可选软件
- Git (用于版本控制)
- VS Code或其他代码编辑器

## 快速启动

### 方法一：使用启动脚本（推荐）

#### 启动后端服务
1. 双击 `start-backend.bat`
2. 脚本会自动检查环境并启动后端服务
3. 服务启动后访问: http://localhost:3000

#### 启动前端项目
1. 双击 `start-frontend.bat`
2. 按提示使用微信开发者工具打开项目
3. 项目路径: `front` 目录

### 方法二：手动启动

#### 启动后端
```bash
cd back/src
npm install
npm start
```

#### 启动前端
1. 打开微信开发者工具
2. 导入项目，选择 `front` 目录
3. 使用测试AppID或填写正式AppID

## 配置说明

### 后端配置
在 `back/src/.env` 文件中配置：

```
DB_HOST=localhost
DB_PORT=3306
DB_NAME=subway_checkin
DB_USER=root
DB_PASSWORD=your_mysql_password
PORT=3000
JWT_SECRET=your-jwt-secret-key
```

### 数据库配置
- **本地MySQL**: 确保MySQL服务已启动
- **云数据库**: 使用云数据库连接信息
- **初始化**: 执行 `mysql-schema.sql` 中的SQL语句初始化数据库

## 常见问题

### 1. 后端启动失败
- 检查Node.js是否安装
- 检查MySQL是否启动
- 检查端口3000是否被占用

### 2. 前端项目无法导入
- 确认微信开发者工具已安装
- 检查项目路径是否正确
- 确认有有效的AppID（可使用测试号）

### 3. 数据库连接失败
- 检查MySQL服务是否启动
- 检查数据库连接配置是否正确
- 检查数据库是否已创建
- 检查用户权限是否正确

### 4. 端口冲突
- 修改 `.env` 文件中的 `PORT` 值
- 或关闭占用3000端口的程序

- 根路径: http://localhost:3000/

## 项目特点

1. **前后端分离**: 微信小程序 + Node.js后端
2. **轻量级**: 使用原生小程序开发，无额外框架
3. **数据持久化**: MongoDB存储用户数据和打卡记录
4. **本地优先**: 支持离线打卡，网络恢复后同步
5. **智能分析**: 提供通勤数据分析和可视化

## 技术支持

如遇问题，请检查：
1. 各软件版本是否符合要求
2. 网络连接是否正常
3. 配置文件是否正确
4. 查看控制台错误信息
