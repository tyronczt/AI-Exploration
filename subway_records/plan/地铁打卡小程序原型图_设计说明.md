
# 地铁打卡小程序原型图_设计说明.md
**文档版本：** 1.0
**最后更新：** 2026年02月07日
**项目状态：** 设计阶段

## 设计需求

### 设计主题
现代、简洁、高效的地铁通勤打卡记录小程序，专注于快速记录通勤关键节点，并通过清晰的数据可视化帮助用户分析出行规律。

### 适配设备
微信小程序移动端，标准尺寸：375px × 812px（基于iPhone尺寸进行设计适配）
布局结构：每个页面遵循 顶部导航栏 + 中央内容区 + 底部导航栏 的基本框架。根据微信小程序设计规范，顶部导航栏（含胶囊按钮）高度约80rpx（转换为px约40px），内容区高度灵活，底部导航栏（Tab Bar）高度约100rpx（转换为px约50px）。本设计说明中的尺寸单位以rpx（小程序响应式像素）为主，开发时需按750rpx设计稿标准进行适配。

### 页面布局结构

**页面1：主页面（打卡页）**
- **顶部导航栏（80rpx）**：
  - 背景色：#1976D2（地铁蓝）
  - 左侧：页面标题“地铁打卡”，白色，字号36rpx，加粗
  - 右侧：设置图标（齿轮形状），白色，点击跳转至设置页面
- **中央内容区（可滚动）**：
  1.  **状态选择器模块（高度约100rpx）**：
      - 位于标题栏下方，背景色白色。
      - 包含一个下拉选择框，宽度占屏幕80%，居中显示。
      - 默认提示文字：“选择当前状态”。
      - 边框：1rpx solid #E0E0E0，圆角10rpx。
  2.  **核心打卡区域（高度约400rpx）**：
      - 页面视觉中心，包含一个大型圆形打卡按钮。
      - 按钮直径：300rpx，背景色为#4CAF50（绿色），代表“行动/记录”。
      - 按钮中央显示当前系统时间（格式：HH:mm），字号48rpx，白色。
      - 按钮下方显示已选中的状态名称，字号32rpx，颜色#333333。
  3.  **最近记录预览模块（高度约200rpx）**：
      - 位于打卡区域下方，背景色#F5F5F5。
      - 模块标题：“最近记录”，字号28rpx，颜色#666666，左对齐。
      - 内容为横向或短列表，展示最近的1-3条打卡记录，包含时间、状态、耗时等关键信息。
      - 点击任意记录可跳转至该记录的详情页。
- **底部导航栏（100rpx）**：
  - 背景色：#FFFFFF，顶部有1rpx的#E0E0E0分割线。
  - 三个Tab项：打卡（当前选中）、记录、分析。
  - 每个Tab包含图标和文字，未选中状态为灰色#999999，选中状态为#1976D2。

**页面2：记录页面**
- **顶部导航栏（80rpx）**：
  - 背景色：#1976D2。
  - 左侧：页面标题“打卡记录”，白色，字号36rpx。
  - 右侧：筛选图标（漏斗形状），白色，点击可弹出时间范围等筛选选项。
- **中央内容区（可滚动，分为上下两部分）**：
  1.  **日历视图模块（固定高度约300rpx）**：
      - 背景色白色。
      - 顶部为月份导航栏，显示左右箭头和当前“YYYY年MM月”。
      - 下方为日历网格（7列×5或6行），显示日期。有打卡记录的日期用一个小圆点（颜色#4CAF50）在日期下方标记。
  2.  **记录列表模块（占据剩余空间）**：
      - 展示所选日期（默认今天）的所有打卡记录。
      - 采用卡片列表形式，每条记录一个卡片，高度约150rpx。
      - 卡片内容左部分显示时间（大字号），右部分显示状态、地点、耗时（小字号）。
      - 卡片底部可放置编辑和删除的图标按钮。
- **底部导航栏（100rpx）**：
  - 同主页面，当前选中“记录”Tab。

**页面3：分析页面**
- **顶部导航栏（80rpx）**：
  - 背景色：#1976D2。
  - 页面标题“通勤分析”，白色，字号36rpx。
- **中央内容区（可滚动）**：
  1.  **关键统计卡片区（高度约200rpx）**：
      - 采用2x2或横向滑动布局，展示核心统计指标，如“本周平均通勤时间”、“最常用路线”、“准时率”等。
      - 每个卡片有图标、数据标题和数值，设计简洁。
  2.  **图表可视化区（高度约300-400rpx）**：
      - 通勤时间趋势图（折线图）：展示最近7天或30天的通勤总时长变化。
      - 各阶段耗时分布图（饼图或柱状图）：分析“家到地铁站”、“地铁内”、“出站到公司”等阶段的平均耗时占比。
  3.  **出行建议卡片区**：
      - 基于分析结果，生成1-2条个性化建议，如“建议您提前5分钟出门，避开周一早高峰”。
- **底部导航栏（100rpx）**：
  - 同主页面，当前选中“分析”Tab。

**辅助页面/组件：**
- **设置页面**：列表形式，包含“账号管理”、“通知设置”、“数据备份与恢复”、“关于我们”等条目。
- **记录详情页/弹窗**：展示单条记录的完整信息，包括精确时间戳、状态、详细位置（地图截图或地址）、耗时、用户添加的备注。提供编辑和删除入口。
- **打卡确认弹窗**：用户点击打卡按钮后弹出，显示即将记录的状态、时间、位置（概要），提供“确认”和“取消”按钮。

### 颜色方案
- **主色调/品牌色**：#1976D2（地铁蓝）。用于顶部导航栏背景、底部导航选中状态、重要按钮和高亮文字。
- **行动色/成功色**：#4CAF50（绿色）。用于核心打卡按钮、成功状态提示、日历打卡标记点。
- **中性色**：
  - 标题文字深灰：#333333
  - 正文文字中灰：#666666
  - 辅助文字/未选中状态浅灰：#999999
  - 分割线/边框色：#E0E0E0
  - 卡片/模块背景浅灰：#F5F5F5
  - 页面背景白色：#FFFFFF
- **状态色**：
  - 警告/提示色：#FF9800（橙色）
  - 错误色：#F44336（红色）

### 排版和字体
- **字体家族**：优先使用微信小程序默认字体（-apple-system, BlinkMacSystemFont, ‘Helvetica Neue’），确保各平台显示一致。
- **字号层级 (rpx)**：
  - 超大标题：48rpx（用于打卡按钮时间显示）
  - 页面主标题：36rpx，加粗（用于导航栏标题）
  - 模块标题/卡片主信息：32rpx
  - 正文：28rpx
  - 辅助信息/标签：24rpx
  - 极小文字：20rpx（如版权信息）
- **字重**：常规体用于正文，加粗体用于标题和关键数据。
- **行高与间距**：文字行高约为字号的1.4-1.6倍。模块间垂直间距统一使用24rpx或32rpx的倍数。

### 组件和元素
- **按钮**：
  - 主按钮（如打卡按钮）：#4CAF50背景，白色文字，300rpx直径圆形，无边框。
  - 次要按钮（如确认弹窗按钮）：#1976D2背景，白色文字，高度80rpx，圆角10rpx。
  - 文字按钮/图标按钮：无背景，使用主色或灰色文字/图标。
- **卡片**：白色背景，圆角12rpx，可添加轻微投影（box-shadow），内边距统一为24rpx。
- **输入框/选择器**：高度80rpx，背景白色，边框1rpx solid #E0E0E0，圆角10rpx，内边距左右20rpx。
- **列表项**：高度不低于100rpx，有底部边框或间距分隔。
- **图标**：采用线性图标风格，线条简洁清晰，关键操作图标需具有高辨识度。大小规范：导航栏图标约40rpx，列表操作图标约32rpx。
- **弹窗**：居中显示，半透明蒙层背景，内容区白色背景，圆角20rpx，有标题、内容和操作按钮区。

### 交互效果
- **基本原则**：遵循微信小程序轻量、即用即走的交互理念，操作路径简洁。
- **核心交互流程**：
  1.  打卡流程：选择状态 -> 点击打卡按钮 -> 确认弹窗 -> 记录成功（按钮可有轻微缩放动画反馈）-> 最近记录更新。
  2.  查看记录：点击底部Tab或日历日期 -> 加载并显示记录列表 -> 点击单条记录 -> 进入详情页。
  3.  分析查看：进入分析页 -> 图表数据加载与渲染 -> 可点击图表部分查看详细数据。
- **动画**：仅用于必要的反馈，如按钮点击态、页面切换过渡、数据加载骨架屏，避免复杂动画影响性能。

### 边界处理
- 本设计为微信小程序，运行在微信App内，无需额外添加模拟设备边框。
- 页面内容区背景色（通常为白色或浅灰#F5F5F5）需与微信App本身的背景色有足够区分，确保内容清晰可辨。
- 所有页面需适配不同尺寸的移动设备屏幕，使用rpx单位并充分测试。

### 连贯性规范
- **统一元素**：
  1.  **底部导航栏**：在所有三个主页面（打卡、记录、分析）中保持完全一致的位置、样式和交互逻辑。
  2.  **顶部导航栏**：高度、背景色、标题字体颜色和大小在所有页面保持一致。左侧逻辑可变化（返回按钮或标题），右侧为当前页面的功能入口。
  3.  **颜色与字体系统**：严格遵循前述颜色方案和排版规范，确保全平台视觉统一。
  4.  **圆角与间距**：卡片圆角、按钮圆角、模块间距等设计Token保持一致。
- **差异化元素**：
  1.  **打卡页**：突出核心的圆形打卡按钮和状态选择器。
  2.  **记录页**：突出日历视图和纵向记录列表，强调时间线。
  3.  **分析页**：突出数据卡片和统计图表，强调信息可视化。
- **视觉一致性**：通过统一的色彩、图标风格、组件样式和布局节奏，确保用户在不同功能模块间切换时获得连贯的体验。
