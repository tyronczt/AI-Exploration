# 地铁打卡记录小程序技术架构文档

## 项目概述

地铁打卡记录小程序是一个基于微信小程序平台开发的通勤记录应用，帮助用户记录每日通勤过程中的各个关键时间节点，并提供智能分析功能。

## 技术架构

### 整体架构

```
┌─────────────────────────────────────────┐
│            微信小程序客户端             │
│  (WXML + WXSS + JavaScript + 小程序API) │
└─────────────────┬───────────────────────┘
                  │
                  │ HTTP API
                  │
┌─────────────────▼───────────────────────┐
│           后端服务 (Node.js)            │
│         Express + MySQL + JWT           │
└─────────────────┬───────────────────────┘
                  │
                  │
┌─────────────────▼───────────────────────┐
│              MySQL数据库                │
│        存储用户信息、打卡记录等          │
└─────────────────────────────────────────┘
```

### 前端技术栈

- **框架**: 微信小程序原生开发
- **语言**: JavaScript (ES6+)
- **样式**: WXSS (类CSS)
- **模板**: WXML (类HTML)
- **状态管理**: 微信小程序内置状态管理
- **本地存储**: 微信小程序Storage API
- **网络请求**: 微信小程序wx.request API

### 后端技术栈

- **运行环境**: Node.js
- **框架**: Express.js
- **数据库**: MySQL (使用Sequelize ORM)
- **认证**: JWT (JSON Web Token)
- **中间件**: 
  - CORS (跨域资源共享)
  - body-parser (请求体解析)
  - dotenv (环境变量管理)
- **开发工具**: nodemon (开发环境热重载)

### 数据模型

#### 核心实体

1. **用户 (User)**
   - 基本信息：昵称、头像、性别等
   - 认证信息：openId、unionId等
   - 配置信息：偏好设置

2. **打卡记录 (Checkin)**
   - 关联用户
   - 关联站点
   - 状态信息：家→地铁站、地铁站→上车等
   - 时间信息：时间戳、日期
   - 位置信息：经纬度、地址
   - 耗时：与前一个状态的间隔
   - 备注信息
   - 成就解锁

3. **站点 (Station)**
   - 站点基本信息：名称、线路、位置
   - 站点特色：介绍、图片等

4. **成就 (Achievement)**
   - 成就名称和描述
   - 解锁条件
   - 图标信息

### API 接口设计

#### 用户相关接口
- `POST /api/users/register` - 用户注册
- `POST /api/users/login` - 用户登录
- `GET /api/users/profile` - 获取用户信息
- `PUT /api/users/profile` - 更新用户信息

#### 打卡相关接口
- `POST /api/checkins` - 创建打卡记录
- `GET /api/checkins` - 获取打卡记录列表
- `GET /api/checkins/:id` - 获取单个打卡记录
- `PUT /api/checkins/:id` - 更新打卡记录
- `DELETE /api/checkins/:id` - 删除打卡记录

#### 站点相关接口
- `GET /api/stations` - 获取站点列表
- `GET /api/stations/:id` - 获取单个站点详情

#### 分析相关接口
- `GET /api/analysis/summary` - 获取统计摘要
- `GET /api/analysis/timeline` - 获取时间线数据
- `GET /api/analysis/stats` - 获取统计数据

### 数据库设计

#### MySQL Tables

1. **users** - 用户表
2. **checkins** - 打卡记录表
3. **stations** - 地铁站点表
4. **achievements** - 成就表
5. **user_preferred_stations** - 用户偏好站点关联表
6. **user_achievements** - 用户成就关联表

#### 索引设计

- 打卡记录按用户和日期索引：`INDEX idx_user_date (user_id, date)`
- 打卡记录按站点索引：`INDEX idx_station (station_id)`
- 用户按openId索引：`UNIQUE KEY open_id (open_id)`

## 技术选型理由

1. **微信小程序**: 轻量级、无需安装、即用即走，适合通勤记录场景
2. **Node.js + Express**: 轻量级后端框架，开发效率高，社区生态丰富
3. **MySQL**: 关系型数据库，数据一致性好，适合复杂查询和统计分析
4. **JWT**: 无状态认证，适合移动端应用
5. **本地存储优先**: 减少网络依赖，提升用户体验
